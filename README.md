# ATIVIDADE_Singleton

**1.1-** Em um sistema onde você precisa de apenas uma conexão com o banco de dados. Surge a necessidade de usar o padrão **Singleton**.

**1.2-** Por que?  
Porque criar várias conexões ao longo do aplicativo pode causar problemas de desempenho e sincronização.  
Com o padrão Singleton, você garante que essa conexão seja **única, unificada e controlada de forma central**.
